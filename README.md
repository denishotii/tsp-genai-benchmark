# TSP GenAI Benchmark

**Can Generative AI (LLMs) reliably implement complex combinatorial optimization
algorithms — and which model does it better?**

This project uses the **Traveling Salesman Problem (TSP)** as a testbed. Three
classical solvers — Greedy nearest-neighbor, Simulated Annealing (SA), and a
Genetic Algorithm (GA) — are implemented two ways and benchmarked on the same
TSPLIB instances:

- **Track A — Reference:** hand-coded implementations following the literature
  (the quality baseline).
- **Track B — GenAI:** the same algorithms generated by **GPT-5.5** and
  **Claude Opus 4.7** across four prompting strategies, then run on the same
  instances.

Benchmark instances (TSPLIB, `EUC_2D`): `eil51`, `berlin52`, `ch130`, `d198`,
`pr439`, `pr1002` — a six-step size ladder from 51 to 1002 cities.

Seminar project — Denis Hoti, KU Ingolstadt, SS 2026.
Full spec: [CLAUDE.md](CLAUDE.md). Code-quality findings:
[paper/code_quality_assessment.md](paper/code_quality_assessment.md).

## Setup

Requires **Python 3.10+** (developed on 3.12).

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

For the GenAI experiments (Track B) you also need API keys:

```bash
cp .env.example .env
# then edit .env and fill in:
#   OPENAI_API_KEY=sk-...
#   ANTHROPIC_API_KEY=sk-ant-...
```

`.env` is gitignored — your keys are never committed. The reference algorithms,
benchmark, and tests need **no** API keys.

## Usage

All commands are run from the project root with the virtualenv active.

### Run the tests

```bash
pytest
```

Covers the utilities, all three reference algorithms (including SA/GA
quality-target regression checks), and the GenAI harness (code extraction,
the sandboxed executor, and offline orchestrator logic — no API calls).

### Run the reference benchmark (Track A)

```bash
python -m src.benchmark.run_benchmark
```

Runs Greedy, SA, and GA on all four instances and writes
`results/benchmark_results.csv` (length, gap-to-optimal, runtime).

### Run the GenAI experiments (Track B)

Needs API keys in `.env`. The full matrix is 2 models × 4 strategies ×
3 algorithms; `--runs N` repeats each cell `N` times to measure variance.

```bash
# Smoke-test one cell (cheap — verifies keys work):
python -m genai_experiments.run_experiments --models claude --strategies zero_shot --algorithms greedy

# Full matrix, 3 runs per cell:
python -m genai_experiments.run_experiments --runs 3

# Offline pipeline test — no API calls, uses a canned response:
python -m genai_experiments.run_experiments --dry-run
```

Each generation's prompt, raw response, and extracted code are saved under
`genai_experiments/generated_code/{model}/{strategy}/run{N}/`, and per-instance
results are appended to `genai_experiments/experiment_log.csv`. Generated code
runs in an isolated subprocess with a wall-clock timeout (default 60 s).

Flags: `--models`, `--strategies`, `--algorithms` (filter the matrix),
`--runs` (repeats per cell), `--timeout` (per-run seconds),
`--max-rounds` (iterative-refinement rounds), `--dry-run`.

### Run the analysis (Phase 4)

The analysis has two pieces — the notebook for interactive exploration and
`make_figures.py` for the publication-quality figure set used in the paper.

```bash
# Interactive notebook (writes comparison_table.csv):
jupyter notebook notebooks/analysis.ipynb

# Or run it headlessly:
jupyter nbconvert --to notebook --execute --inplace notebooks/analysis.ipynb

# Generate the publication figures (used in the paper):
python -m genai_experiments.make_figures
```

The notebook produces `results/comparison_table.csv`. `make_figures.py`
writes nine figures to `results/figures/` (gap by algorithm × model with
error bars, strategy heatmap, gap by instance size, reliability, LLM vs.
reference, strategy variance, plus the explanatory tour comparison,
pipeline, and 2-opt diagrams).

## Algorithms (reference implementations)

| Solver | Key choices | Typical gap-to-optimal |
|---|---|---|
| **Greedy** | nearest-neighbor, single start (city 0) | 16–28% |
| **Simulated Annealing** | 2-opt neighborhood, geometric cooling (α=0.995), median-calibrated initial temperature, L=2n Markov chain, greedy warm start | 0–10% (≤5.5% up to 198 cities; ~10% at 439–1002) |
| **Genetic Algorithm** | permutation encoding, Order Crossover, swap mutation, tournament selection (k=5), elitism; *pure* GA (no local search) | 11–915% (collapses at scale — the design exposes what LLMs add) |

Distances use the **TSPLIB `EUC_2D` convention** (`floor(sqrt(dx²+dy²) + 0.5)`,
nearest-integer rounding), so gap-to-optimal is computed exactly against the
published optima.

## Reproducibility

- Reference algorithms seed their RNGs (`seed=42` by default) — deterministic.
- LLM generation is inherently non-deterministic; the harness logs every full
  response, and `--runs N` quantifies the variance.
- The analysis notebook regenerates all tables/figures from the committed
  `experiment_log.csv`.

## TSPLIB data provenance

All six benchmark instances in `data/tsplib/` were downloaded from the
**Rice University softlib mirror** of TSPLIB95:

```
http://softlib.rice.edu/pub/tsplib/tsp/<name>.tsp.gz
```

- `eil51`, `berlin52`, `ch130`, `d198` — downloaded 2026-05-24
- `pr439`, `pr1002` — added 2026-06-24 to extend the size ladder
  to ~1000 cities where pure-GA fully collapses and LLM-generated
  code-scaling differences become visible

The canonical TSPLIB95 distribution is hosted by Reinelt at Heidelberg
(`comopt.ifi.uni-heidelberg.de/software/TSPLIB95/`), but that host was
unreachable at the original download time. Rice's softlib is a long-established
academic mirror of the same files. All files use `EDGE_WEIGHT_TYPE: EUC_2D`,
and the published optima are: `eil51=426`, `berlin52=7542`, `ch130=6110`,
`d198=15780`, `pr439=107217`, `pr1002=259045`.
